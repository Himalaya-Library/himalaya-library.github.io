1 # Himalaya
2 
3 ![](https://img.shields.io/github/v/release/Himalaya-Library/Himalaya)
4 [![Build Status](https://github.com/Himalaya-Library/Himalaya/workflows/test/badge.svg)](https://github.com/Himalaya-Library/Himalaya/actions)
5 
6 Himalaya calculates three-loop corrections of order O((αt + αb)*αs^2)
7 to the MSSM CP-even Higgs mass matrix and to the quartic Higgs
8 coupling λ in the DR'-bar scheme using the results of:
9 
10 * R. V. Harlander, P. Kant, L. Mihaila and M. Steinhauser, *Higgs
11  boson mass in supersymmetry to three loops*, [*Phys. Rev. Lett.*
12  **100** (2008) 191602](https://doi.org/10.1103/PhysRevLett.100.191602),
13  [[0803.0672](https://arxiv.org/abs/0803.0672)],
14 
15 * P. Kant, R. V. Harlander, L. Mihaila and M. Steinhauser, *Light MSSM
16  Higgs boson mass to three-loop accuracy*, [*JHEP* **08** (2010)
17  104](https://doi.org/10.1007/JHEP08(2010)104),
18  [[1005.5709](https://arxiv.org/abs/1005.5709)].
19 
20 Please refer to these papers as well as
21 
22 * R. V. Harlander, J. Klappert and A. Voigt, *Higgs mass prediction in
23  the MSSM at three-loop level in a pure DR context*, [*Eur.Phys.J.*
24  **C77** (2017) no.12, 814](https://doi.org/10.1140/epjc/s10052-017-5368-6),
25  [[1708.05720](https://arxiv.org/abs/1708.05720)],
26 
27 * R. V. Harlander, J. Klappert, A. D. Ochoa Franco and A. Voigt,
28  *The light CP-even MSSM Higgs mass resummed to fourth logarithmic order*,
29  [*Eur.Phys.J.* **C78** (2018) no.10, 874](https://doi.org/10.1140/epjc/s10052-018-6351-6),
30  [[1807.03509](https://arxiv.org/abs/1807.03509)]
31 
32 * R. V. Harlander, J. Klappert, and A. Voigt,
33  *The light CP-even MSSM Higgs mass including N3LO+N3LL QCD corrections*,
34  [*Eur.Phys.J.* **C80** (2020) no.3, 186](https://doi.org/10.1140/epjc/s10052-020-7747-7),
35  [[1910.03595](https://arxiv.org/abs/1910.03595)]

37 when using Himalaya.

39 ## Requirements
40 
41 Himalaya requires:
42 * C++ compiler
43 * FORTRAN compiler
44 * [CMake](https://cmake.org/) >= 3.1
45 * [Eigen3](http://eigen.tuxfamily.org)
46 
47 ## Building Himalaya
48 
49 ### Installation of dependencies
50 
51 The required Eigen library can be installed using the package manager
52 of your Linux distribution. On Debian/Ubuntu, for example, one may
53 run:
54 
55 ```sh
56 sudo apt-get install libeigen3-dev
57 ```
58 
59 Alternatively, the [Conan](https://conan.io/) package manager can be
60 used to install the dependencies:
61 
62 ```sh
63 mkdir -p build
64 cd build
65 conan install ..
66 ```
67 
68 ### Compilation of Himalaya
69 
70 Himalaya uses CMake to generate files for build automation. To build
71 Himalaya one should first create a separate `build` directory inside
72 Himalaya's top directory. Afterwards, `cmake` should be called:
73 
74 ```sh
75 mkdir -p build
76 cd build
77 cmake ..
78 ```
79 
80 `cmake` will search for required dependencies on your system, see the
81 previous section. If you want to use the dependencies installed by
82 Conan (see above), you need to run instead:
83 
84 ```sh
85 cmake .. -DCMAKE_TOOLCHAIN_FILE=./conan_paths.cmake
86 ```
87 
88 After calling `cmake` the build directory contains all required build
89 files. Assuming that GNU make is used, one can start the build by running
90 
91 ```sh
92 make
93 ```
94 
95 By default the example executable `example` is created from
96 [examples/example.cpp](examples/example.cpp), which prints all loop
97 corrections calculated by Himalaya for a given MSSM parameter point:
98 
99 ```sh
100 ./example
101 ```

103 ## Running Himalaya
104 
105 When the build is complete, the libraries `libDSZ` and `libHimalaya`
106 have been created. The latter must be linked to user-written programs
107 to call the routines of Himalaya. The library `libDSZ` is optional
108 and has to be linked in addition, if the program does not already
109 incorporate the associated FORTRAN code of P. Slavich for the 2-loop
110 corrections [[hep-ph/0105096](https://arxiv.org/abs/hep-ph/0105096),
111 [hep-ph/0112177](https://arxiv.org/abs/hep-ph/0112177),
112 [hep-ph/0212132](https://arxiv.org/abs/hep-ph/0212132),
113 [hep-ph/0305127](https://arxiv.org/abs/hep-ph/0305127)].
114 
115 ### C++ interface
116 
117 We present a brief step by step guide how to run Himalaya at the C++
118 level and obtain the three-loop corrections to the CP-even Higgs mass
119 matrix in the MSSM in the DR'-bar scheme and the quartic Higgs
120 coupling of the Standard Model in the MS-bar scheme.
121 
122 First one has to include the header
123 
124 ```cpp
125 #include "himalaya/HierarchyCalculator.hpp"
126 ```
127 
128 in the C++ file. The MSSM DR'-bar parameters must be stored in a
129 `Parameters` object. Here, an example for the SPS1a benchmark point
130 is given:
131 
132 ```cpp
133 himalaya::Parameters pars; // DR'-bar parameters struct
134 pars.scale = 4.67491329E+02; // renormalization scale
135 pars.mu = 3.52600579E+02; // mu parameter
136 pars.g3 = 1.09949966E+00; // gauge coupling g3 SU(3)
137 pars.vd = 2.49832484E+01; // VEV of down Higgs doublet
138 pars.vu = 2.43650538E+02; // VEV of up Higgs doublet
139 pars.mq2 << 2.99793928E+05, 0, 0,
140  0, 2.99792102E+05, 0,
141  0, 0, 2.49327504E+05; // soft-breaking squared left-handed squark mass parameters
142 pars.md2 << 2.78275669E+05, 0, 0,
143  0, 2.78273780E+05, 0,
144  0, 0, 2.74928741E+05; // soft-breaking squared right-handed down-squark mass parameters
145 pars.mu2 << 2.80477426E+05, 0, 0,
146  0, 2.80475621E+05, 0,
147  0, 0, 1.80478484E+05; // soft-breaking squared right-handed up-squark mass parameters
148 pars.Ad << 0, 0, 0,
149  0, 0, 0,
150  0, 0, -784.3356416708631; // trilinear down type squark-Higgs coupling matrix
151 pars.Au << 0, 0, 0,
152  0, 0, 0,
153  0, 0, -527.8746242245387; // trilinear up type squark-Higgs coupling matrix
154 pars.MA = 3.92960954E+02; // Mass of the CP-odd Higgs
155 pars.MG = 5.88220143E+02; // Mass of the Gluino
156 pars.MW = 8.04136643E+01; // Mass of the W boson
157 pars.MZ = 9.06817306E+01; // Mass of the Z boson
158 pars.Mt = 1.52117491E+02; // Mass of the top quark
159 pars.Mb = 2.42010269E+00; // Mass of the bottom quark
160 pars.Mtau = 1.777; // Mass of the tau lepton
161 ```
162 
163 Afterwards one can create a `HierarchyCalculator` object for the
164 chosen parameter point:
165 
166 ```cpp
167 himalaya::HierarchyCalculator hc(pars);
168 ```
169 
170 To calculate the loop corrections in the DR'-bar scheme one needs to
171 call:
172 
173 ```cpp
174 // the boolean argument switches between corrections proportional to αt (false) or αb (true)
175 himalaya::HierarchyObject ho = hc.calculateDMh3L(false);
176 ```
177 
178 All information which has been gathered during the calculation will be
179 stored in the returned `HierarchyObject` and can be accessed by member
180 functions.
181 
182 To extract the three-loop correction to the Higgs mass matrix one
183 needs to call:
184 
185 ```cpp
186 // returns a 2x2 matrix with the αt*αs^2 correction for the given parameter point
187 auto dMh3L = ho.getDMh(3);
188 ```
189 
190 To extract the three-loop correction to the quartic Higgs coupling λ of
191 the Standard Model in the DR'-bar scheme in the convention of
192 [[1407.4081](https://arxiv.org/abs/1407.4081)] one needs to call
193 
194 ```cpp
195 double delta_lambda_3L = ho.getDLambda(3);
196 ```
197 
198 The three-loop shift to the MS-bar scheme, ocurring when the one- and
199 two-loop corrections are expressed in terms of the Standard Model
200 MS-bar strong gauge and top Yukawa couplings can be obtained by
201 calling `ho.getDLambdaDRbarPrimeToMSbarShift(3)`.
202 
203 An uncertainty estimate of the calculated three-loop λ due to the
204 truncation of the mass hierarchy expansions can be obtained by calling
205 
206 ```cpp
207 double delta_lambda_3L_uncertainty = ho.getDLambdaUncertainty(3);
208 ```
209 
210 A full and detailed example can be found in
211 [examples/example.cpp](examples/example.cpp).
212 
213 **Example**:
214 
215 ```cpp
216 #include "himalaya/HierarchyCalculator.hpp"
217 #include <iostream>
218 #include <cmath>
219 
220 himalaya::Parameters setup_point(double MS, double tb, double xt)
221 {
222  himalaya::Parameters pars;
223 
224  const double MS2 = MS*MS;
225  const double Xt = xt*MS;
226  const double beta = std::atan(tb);
227  pars.scale = MS;
228  pars.mu = MS;
229  pars.g1 = 0.46;
230  pars.g2 = 0.65;
231  pars.g3 = 1.166;
232  pars.vd = 246*std::cos(beta);
233  pars.vu = 246*std::sin(beta);
234  pars.mq2 << MS2, 0, 0,
235  0, MS2, 0,
236  0, 0, MS2;
237  pars.md2 << MS2, 0, 0,
238  0, MS2, 0,
239  0, 0, MS2;
240  pars.mu2 << MS2, 0, 0,
241  0, MS2, 0,
242  0, 0, MS2;
243  pars.ml2 << MS2, 0, 0,
244  0, MS2, 0,
245  0, 0, MS2;
246  pars.me2 << MS2, 0, 0,
247  0, MS2, 0,
248  0, 0, MS2;
249  pars.Au << 0, 0, 0,
250  0, 0, 0,
251  0, 0, Xt + pars.mu/tb;
252  pars.Ad << 0, 0, 0, 0, 0, 0, 0, 0, 0;
253  pars.Ae << 0, 0, 0, 0, 0, 0, 0, 0, 0;
254  pars.Yu << 0, 0, 0, 0, 0, 0, 0, 0, 0.862;
255  pars.Yd << 0, 0, 0, 0 ,0 ,0 ,0 ,0, 0.133;
256  pars.Ye << 0, 0, 0, 0, 0, 0, 0, 0, 0.101;
257  pars.MA = MS;
258  pars.M1 = MS;
259  pars.M2 = MS;
260  pars.MG = MS;
261 
262  pars.validate(true);
263 
264  return pars;
265 }
266 
267 int main()
268 {
269  const auto point = setup_point(2000., 20., std::sqrt(6.));
270  himalaya::HierarchyCalculator hc(point);
271 
272  try {
273  // calculate the 3-loop corrections O(α_t*α_s^2)
274  const auto ho = hc.calculateDMh3L(false);
275 
276  // extract 2x2 matrix with three-loop O(αt*αs^2) corrections
277  const auto dMh3L = ho.getDMh(3);
278  // extract three-loop O(αt*αs^2) correction to λ (DR'-bar scheme)
279  const double delta_lambda_3L_DR = ho.getDLambda(3);
280  // extract uncertainty estimate
281  const double delta_lambda_3L_uncertainty = ho.getDLambdaUncertainty(3);
282  // convert to MS-bar scheme
283  const double delta_lambda_3L_MS =
284  delta_lambda_3L_DR + ho.getDLambdaDRbarPrimeToMSbarShift(3);
285 
286  std::cout << "Δλ(3-loop,DR') = " << delta_lambda_3L_DR
287  << " +- " << delta_lambda_3L_uncertainty << '\n'
288  << "Δλ(3-loop,MS) = " << delta_lambda_3L_MS
289  << " +- " << delta_lambda_3L_uncertainty << '\n';
290  } catch (const std::exception& e) {
291  std::cerr << e.what() << '\n';
292  }
293 
294  return 0;
295 }
296 ```
297 
298 Output:
299 
300 ```
301 Himalaya info: Δλ(3-loop,DR') = 0.000315613 +- 0.00203118
302 Himalaya info: Δλ(3-loop,MS) = -0.000455415 +- 0.00203118
303 ```

305 ### Mathematica interface
306 
307 Since version 2.0.0 Himalaya can be run from within Mathematica using
308 the LibraryLink interface. To load Himalaya into a Mathematica
309 session, first, the file
310 [source/himalaya/LibraryLink/Himalaya_LibraryLink.m](source/himalaya/LibraryLink/Himalaya_LibraryLink.m)
311 must be loaded, which defines the Himalaya's Mathematica interface
312 functions. Assuming the current directory is the `build/`
313 sub-directory of Himalaya, loading `Himalaya_LibraryLink.m` may be
314 done by calling
315 
316 ```.m
317 Get[FileNameJoin[{"..", "source", "himalaya", "LibraryLink", "Himalaya_LibraryLink.m"}]];
318 ```
319 
320 Afterwards, the LibraryLink `Himalaya_LibraryLink.so` must be loaded
321 using the `InitializeHimalaya[]` function:
322 
323 ```.m
324 InitializeHimalaya[FileNameJoin[{".", "Himalaya_LibraryLink.so"}]];
325 ```
326 
327 After the initialization, the function `HimalayaCalculateDMh3L[]` is
328 available, which calculates the loop corrections implemented in
329 Himalaya. A full and detailed example can be found in
330 [examples/example.m](examples/example.m).
331 
332 **Example**:
333 
334 ```.m
335 Get[FileNameJoin[{"..", "source", "himalaya", "LibraryLink", "Himalaya_LibraryLink.m"}]];
336 
337 InitializeHimalaya[FileNameJoin[{".", "Himalaya_LibraryLink.so"}]];
338 
339 MS = 2000;
340 TB = 20;

343 result = HimalayaCalculateDMh3L[

352  g1 -> 0.46,

363  {0,0,0},
364  {0,0, Xt + MS/TB }},
365  Ad -> 0 IdentityMatrix[3],
366  Ae -> 0 IdentityMatrix[3],

368  {0,0,0},
369  {0,0, 0.862 }},
